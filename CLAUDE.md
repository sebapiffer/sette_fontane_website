# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

One-page showcase site for **Sette Fontane**, an organic winery in Giovo (Val di Cembra, Trentino). First wine: San Florian, a Müller Thurgau in two formats (Renana 0.75 L, Magnum 1.5 L). The original spec is `instructions.md`; brand source assets (logo PDF, label, bottle photos) live in `references/`. UI copy and comments are in Italian.

## Commands

```bash
npm run dev       # Vite dev server
npm run build     # production build to dist/
npm run preview   # serve the production build
```

There is no lint or test tooling configured.

## Architecture

Vite + React 18 + Tailwind v3, animations via GSAP (ScrollTrigger + Flip) through the `useGSAP` hook from `@gsap/react`. Icons from `lucide-react`.

- `src/App.jsx` — registers GSAP plugins once and stacks the page sections in order: Preloader, Navbar, Hero, Azienda, ChiSiamo, Territorio, IntroVini, SanFlorian, ComingSoon, Footer.
- `src/components/IntroVini.jsx` — the "La nostra cantina" separator (`id: la-nostra-cantina`; no nav link points at it — the menu jumps straight to `#san-florian`): a full-bleed band, long and low, in solid `antracite`. Its whole job is the cut, so it breaks every habit of the other sections on purpose — no photographic background, no scrim, no columns, no CTA, no `SfondoSezione`. It was `moro` until the client read that band as "too brown and not cohesive", and the diagnosis stuck: the home's only grounds are `antracite` and `creta`/`offwhite` with `tortora` as an accent, so `moro` — a ground used nowhere else — read as foreign rather than as a deliberate caesura. On `antracite` the band is now the **opening of the dark chapter** instead of a stripe of its own: the top edge still cuts hard against Territorio's offwhite, but the ground runs on into San Florian, so the colour alone no longer closes it. The bottom cut is made by `.vini-chiusura`, a full-bleed tortora hairline — the same accent as the hero's rule under the title. It must **not** be given the `.vini-filo` class: the Viticcio anchors its crossing nodes to those elements *by index*, and a third one would shift them. The drops logo sits *inside* the title, between "La nostra" and "cantina" (`vini.titleParts` in `content.js`): it's the band's only vertical element and it sets the height. It is **perfectly centred on the viewport**, and that is structural, not tuned: the two title halves are each `flex-1 basis-0`, so they measure the same regardless of the tronconi being different widths ("LA NOSTRA" > "CANTINA") — the text hugs the logo, the hairline takes the remainder out to the edge. Don't "fix" the centring by nudging margins; if it drifts, a half has stopped being `basis-0`. Below `md` the size, tracking and gap all step down (`tracking-[0.16em] md:tracking-[0.28em]`, and the `-mr` trim tracks it): at 360 px the full title doesn't fit, and what gets squeezed out is the hairlines — they collapsed to **width 0**, taking the Viticcio's channel with them. Those numbers were measured across 360/390/430/768/1024/1440/1920 with a Playwright scratch script, not eyeballed; the left hairline (the mobile anchor) bottoms out at ~14 px (it was ~17 until `DropsLogo` was redrawn on the PDF geometry, which is wider at the same height — any further widening of the logo eats this margin directly). The title is `font-bold` — Mason Sans ships a real 700 face (`MasonSans-Bold.woff2`), so this is not synthetic. Verify weight by **measuring**, never by looking at a downscaled screenshot: a full-width capture shrunk to view thins the strokes enough to look like 400. The reliable check is that the rendered word's width matches a forced-`700` span at the same size and tracking (at 46.4 px, "La" is 81.98 px at 700 vs 79.98 at 400) — an hour was lost chasing a bold that was there all along. Any further growth of the title or logo has to be re-measured against that floor; the drops fall in with `CADUTA_GOCCE`, the same cadence as the preloader and hero. The visible title is `aria-hidden` — split in two around a logo it doesn't read as a sentence — and an `sr-only` h2 carries the real text. The two hairlines are `.vini-filo`, and they are **load-bearing for the Viticcio**: each one starts exactly where the title ends and runs to the edge, i.e. its rect *is* the free channel beside the text, which is why the tendril's crossing nodes anchor to `.vini-filo` (index 1 = right, desktop; index 0 = left, mobile) instead of to fractions of the section — the channel then tracks the title's width by itself at any viewport, no retuning.
- `src/components/Preloader.jsx` — full-screen loading overlay with the drops logo looping (same fall-in cadence as the hero opening). It waits for window `load` + `document.fonts.ready` (min 1.6 s, max 6 s), fades out, unmounts, then sets `document.documentElement.dataset.ready` and dispatches `sf:ready` and a `ScrollTrigger.refresh()`. **The hero opening timeline is paused and starts on `sf:ready`** — any new mount-time entrance animation should gate on the same signal or it will play behind the preloader.
- `src/data/content.js` — **all copy lives here**, never hardcoded in components. Texts marked `[placeholder]` await final copy from the client; the "L'azienda" copy is real (taken from the packaging carton). Keep the structure intact when editing.
- `src/hooks/useReveal.js` — shared scroll-reveal used by every section: elements tagged `data-reveal` fade up, `data-reveal-img` uncover via clip-path. Wrapped in `gsap.matchMedia` so `prefers-reduced-motion` disables it. New sections should reuse this hook rather than writing their own reveal.
- `src/components/SfondoSezione.jsx` — full-bleed photographic section background (Azienda, ChiSiamo, Territorio, SanFlorian, Footer; ComingSoon and IntroVini are deliberately left clean — ComingSoon is the pause before the footer and the end of the Viticcio, IntroVini is a flat colour cut and a photo would soften exactly the edge it exists to make). Decorative only (`aria-hidden`, empty `alt` — the informative photos stay in the figures), still (no parallax: images don't move on this site), **desaturated** (`grayscale(1) sepia(0.22) contrast(1.05)`, overridable via the `filtro` prop — the color comes from the brand scrim above, not from the photo), and always under a **scrim passed as `children`**: the scrim is tuned per section on where the text actually falls (full opacity over the text column, opening up over the figure; vertical instead of horizontal below `md`, where the columns stack). Readability comes first — worst case (a solid black photo under the light sections) still clears 4.5:1. Two stacking rules, both about the Viticcio: the host section must be `relative overflow-hidden` but must **not** create a stacking context (no `isolate`, no z-index), or the `z-10` figures stop sitting in front of the tendril and the weave is lost; and the content container must be `relative` and come *after* the background in the DOM, or the positioned background paints over the in-flow text. The tendril, which comes after the sections in the wrapper, correctly runs *over* these backgrounds.
- `src/components/Navbar.jsx` — fixed navbar that switches light/dark styling based on which section is under it; every section declares `data-nav-theme="dark"` or `"light"`. Any new section must set this attribute.
- `src/components/SanFlorian.jsx` — the most delicate component: a GSAP Flip transition zooms the bottle into a full-screen overlay (dialog) with tasting notes and a format selector. Scroll is locked via `document.body.style.overflow` while open. Below `lg` the zoomed bottle is fixed at the top and fades out as the overlay scrolls (`onOverlayScroll`) so it never covers the notes; its opacity is restored in `toggleZoom(false)` before the return Flip state is captured. While the overlay is active it sets `document.documentElement.dataset.sfZoom` (from open through close): the zoomed bottle goes `position: fixed`, leaves the flow and shortens the page, which would fire the Viticcio's ResizeObserver and rerun its (heavy) path regen on the main thread mid-Flip — the flag makes Viticcio skip that; `toggleZoom(false)`'s Flip `onComplete` clears the flag and dispatches `sf:relayout` so Viticcio recomputes once, settled.
- `src/components/Viticcio.jsx` — decorative tendril line (tortora, the logo color) drawn via `stroke-dashoffset` scrubbed with ScrollTrigger over all post-hero sections (they sit in a `relative` wrapper in `App.jsx`). The path is **generated at runtime**, not hand-authored: each breakpoint config lists nodes anchored to real page elements (fractions of an anchor's rect, `p(selector, fx, fy, {dy, asola?, ramo?, foglia?}, index?)`), joined by a **centripetal** Catmull-Rom spline that passes exactly through every node with smooth, fluid curves (no meander/ondulation — the line is deliberately restrained) — so the line adapts by itself to width changes and section-height variations (re-measured on `sf:ready`, window resize, and a ResizeObserver on the wrapper; all measures are skipped until `sf:ready` — before that the preloader covers everything and fonts/images still in flight would trigger wasted regens that steal main thread from the loading animation). The centripetal parameterization is load-bearing: the point list mixes waypoints hundreds of px apart with spiral samples a few px apart, and uniform Catmull-Rom kinks at those junctions. A node with `asola: {raggio, lato}` splices a sequence of **exact circular arcs** (`A` commands, not sampled points — the loop is geometrically perfect at every size) entered **tangent to the direction of travel** (estimated from the neighbouring nodes, or fixed via `direzione`): one full turn (half at full radius, half at 0.8) that re-emerges *inside* the ring already heading in the original travel direction, so the continuation crosses the ring exactly once at a sharp angle — a closed pencil-loop eyelet with no doubled tracts; the spline resumes from the arc's exit with its Bézier handle clamped to the arc's tangent (C1 throughout). The button wrap (`nodoAvvolgimento`) is the same standard eyelet, just anchored on the pill's right cap with a fixed entry direction. A node with `ramo: {lato, punti, figli?}` spawns a separate path that is a **simple bifurcation of the main spline**, not a parametric curve: it grafts tangent to the main line (same C1 saddling as the asole — the Bézier handle is clamped to the tangent) and then runs through its own `punti` array, joined by the exact same centripetal Catmull-Rom spline used for the main line. Each point in `punti` uses the **same page-anchored structure as the main line's nodes**: absolute page coordinates produced by `p(selector, fx, fy, …)` — a fraction of a real anchor's rect, typically the same anchor the branch's own graft node used — rather than the local stem-relative `{u, v}` offsets the branches used before; a branch tracks layout changes the way the main line does, instead of holding a fixed vector off its graft. Tuning a branch means editing those fractions, measured against the DOM like every other node, not an angle/length/radius formula (and not local `u`/`v` numbers either anymore — that scheme survives only for the small, fixed eyelet gesture, see `puntoLocale`/`PUNTI_CAPPIO`). A branch (at any level — top-level `ramo` or a `figli` sub-branch, since both go through the same `generaRamo`) can end in a `ricciolo`, chosen by `ricciolo.tipo`: `'induttore'` (default) — `riccioloInduttore`, a row of **exact-arc loops** (reusing the same eyelet construction as `asola`, factored into `costruisciGiro`) described one by one by a `spire` array, each `{ampiezza, spaziatura?, inclinazione?, lato?}` — `ampiezza` is that loop's radius, `spaziatura` is the straight-but-tangent run travelled *before* drawing it (default `0.9×ampiezza`, the ratio it shipped with before becoming configurable), `inclinazione` (degrees) rotates the tangent before that run so consecutive loops fan out or wave instead of sitting dead in line, and `lato` can flip winding direction loop to loop; it's a separate path grafted on after the stem with the stem's exit tangent clamped as the spline's `tOut` (C1, same trick as the asole). Or `'piatta'` — `puntiSpiralePiatta`, the original sampled shrinking spiral (from before branches moved to explicit points), restored and made configurable (`ampiezza`, `giri`, `restringimento` — the fraction of radius lost per full turn), which returns raw points appended to the stem's own point array *before* the single `splineSegmento` pass, so stem and spiral are one continuous curve with no graft seam (unlike the inductor's `tOut`-clamped join). Branches are deliberately **few but wide**: they live only where, diverging, they find open space or the outer edge of a photo to skirt — never parallel to the line, never hidden behind a `z-10` image, never crossing the line or another branch. `figli` lets a branch **fork**: each child grafts tangent to a point inside the parent's resolved point chain (index `da`, 0 = the graft itself) with the tangent estimated from its neighbours (same criterion as the main line), and has its own `punti`, recursively (`generaRamo` returns an array of paths). All forks are separate `subBranches` paths, so the main line stays single. (`catenaGambo`, the old integrated-curvature stem chain, survives only for the leaf twig — see `foglia` below — since it still needs a `svolta`-driven diverge-then-straight profile there.) Each path is revealed by the scrubbed timeline when the tip passes its graft point: the fraction of the main line at each graft is computed **at generation time** by a sampler that accumulates points and cumulative length while the path is emitted (pure arithmetic, rendered as `data-progresso`) — never by measuring the mounted path with `getPointAtLength`, which cost ~600 DOM calls ≈ 1.2 s of blocked main thread per regen (it froze the preloader mid-animation). The sampler feeds in travel order: spline tratta first, then the asola arcs (see the ordering comment in `generaViticcio` — swapping them double-counts lengths). Group `opacity` (not per-stroke alpha) keeps the grafts invisible. A node with `foglia: {lato, scala, rotazione}` (plus `altezza`/`lunghezza` from the config defaults) grows a **ramoscello**: the exact same stem logic as `ramo` (`catenaGambo`, tangent at the graft, diverging toward `lato`) but short and with no closing curl — it's rendered into `subBranches` alongside the real branches, same class/stroke/reveal treatment — and a vine-leaf outline (the wireframe traced in `references/leaf-wireframe.svg`, embedded verbatim as `FOGLIA_D`, a single stroke path with no fill) is grafted onto its tip. The leaf's petiole point (`FOGLIA_PICCIOLO`, measured — not the shape's bbox center) is anchored EXACTLY on the twig's tip, then rotated so its natural petiole→tip axis (`FOGLIA_ASSE`) continues the twig-tip's own tangent — the leaf carries the twig's motion straight through; `rotazione` adds per-leaf jitter and `scala` (× the config's `altezza` in px, and shared with the twig's own `lunghezza`) varies size — both exist purely so the handful of leaves don't read as one stamped shape. Same stroke weight and color as the tendril itself (`spessoreLinea`, `currentColor`) — stroke width is divided by the leaf's own `scala` before rendering (`spessoreLinea / f.scala`) because the SVG `scale()` used to size the leaf would otherwise scale the stroke too; `scala` is rounded to 3 decimals, not `arrotonda`'s 1 (its natural range is ~0.04–0.1, where 1-decimal rounding collapses every leaf to the same 0.1 and erases the size variation between them). Leaves are few (3 desktop, 2 mobile) and live only in the empty gaps BETWEEN sections, never on a node that already carries an `asola` or `ramo` — this is decoration on the seams, not a new density of detail along the line. Each leaf's outline is dash-revealed by the same scrubbed timeline as the branches, timed via the same generation-time fraction (rendered as `data-progresso` on its `<g>`, computed from the leaf's tip position). Placement was verified at 1024/1440/1920 and 360/390/430 px by a checker (scratch `collisioni.mjs`) that flags three things: line-over-text (`Range.getClientRects` boxes), branch fraction hidden inside image boxes, and branch-vs-branch / branch-vs-line proximity — all driven to zero. On mobile there is no room for the button wrap (Andrea's full-width bio sits between photo and CTA), so the tendril skips it and runs down the right margin — the CTA is `z-10` anyway. The **IntroVini band** is the one place where a section actively pushes the line around: its title is centred and sits exactly where the line used to run, so the crossing nodes anchor to `.vini-filo` (see IntroVini above) and the line detours right of the title on desktop, left of it on mobile, crossing the hairline — a decorative rule, the weave is welcome there. That detour forced two retunings that must stay together: the desktop bottle entry moved from `.sf-bottle -1.02, -0.199` to `-0.02` (the old node sat 18 px below the band, so coming back from the right channel in that gap was an elbow, not a curve), and the mobile leaf under Territorio moved from `fy 1.18` to `1.06` (the band swallowed the gap it lived in and left it straddling the band's edge). Anchor rects are measured via the `offsetTop`/`offsetLeft` chain (immune to the inline transforms GSAP reveals leave around; SVG anchors fall back to bounding-rect deltas minus GSAP's x/y). The line weaves through the layout: *behind* the elements marked `relative z-10` (Azienda figure, ChiSiamo CTA, Territorio figure, the San Florian bottle), *in front* of everything else (e.g. the second ChiSiamo photo), never touching text, ending on the ComingSoon drops logo. Two configs: desktop (≥ `lg`) and mobile (< `md`); in the 768–1023 band the tendril stays hidden (mixed layouts, no node set is safe). Node fractions were tuned against the real DOM: if a section's internal structure changes, retune by measuring, not by eye — but pure resizes now need no retuning. The "wrapping" illusion comes from asole that straddle the edges of `z-10` elements (half the ring hides behind the photo/bottle) and from `nodoAvvolgimento`, which coils around the rounded right cap of pill buttons — place such curls only where the element's visible contour coincides with its box edge, or the line seems to vanish into nothing. Static (fully drawn) with `prefers-reduced-motion`.

## Design system

- Brand palette is defined in `tailwind.config.js`: `antracite` #1D1D1B, `moro` #634E42, `tortora` #A48A7B, plus derived `creta`, `sabbia`, `offwhite`. Use these token names, not raw hex.
- Fonts: brand typefaces self-hosted from `public/fonts/` via `@font-face` in `src/index.css` (sources in `references/Fonts.zip`; two critical woff2 are preloaded in `index.html`). Mason Sans (`font-display`) for the wordmark and titles — it matches the bottle label lettering; LT Wave (`font-sans` and `font-prose`) for everything else. Mason Sans only ships 400/700: `font-medium`/`font-semibold` resolve to 400/700 via CSS font matching, LT Wave also has 300 (`font-light`). No Google Fonts.
- Shared component classes in `src/index.css`: `.eyebrow`, `.btn`, `.btn-dark`, `.btn-light`.
- Fluid typography via `clamp()` (e.g. `text-[clamp(2rem,5vw,4rem)]`), mobile-first layouts (`grid-cols-1 md:grid-cols-2`).
- Recurring formal motifs: round-arch images (`rounded-t-full`), the seven-drops logo rebuilt as SVG in `DropsLogo.jsx`, dark hero fading toward light backgrounds through tortora.
- Images: the figures are the *subjects*, the section backgrounds (`SfondoSezione`) are *depth* — large, quiet, always under a scrim. Territorio's panorama is now the background and its figure shows a detail (porfido terraces), so the same view isn't told twice. Images never move with the scroll: parallax was tried and removed on purpose — reveals are the only motion they get.

## GSAP pitfalls (learned the hard way)

- Inline transforms GSAP leaves on containers break `position: fixed` descendants — that's why `useReveal` uses `clearProps: 'transform'` on text reveals. Keep that in mind when animating anything that wraps a fixed element.
- Any tween that touches the San Florian bottle's transform (today: the format-change crossfade, a `gsap.from` that runs at mount and leaves an inline `translate(0,0)`; formerly also an idle bobbing tween, since removed) leaves inline transforms that override `.bottle-zoom`'s own `translate` — the zoomed bottle ends up half off-screen. That's why `toggleZoom(true)` does `clearProps: 'transform'` BEFORE capturing `Flip.getState`: keep it, and pause any looping transform tween there too (the glow pulse is paused/resumed this way).
- The bottle carries a huge `drop-shadow` (70px blur); recomputing it every frame while the Flip scales the bottle janks the whole zoom. The Flip therefore turns the filter off in `onStart` (`filter: 'none'`) and restores it in `onComplete` (`clearProps: 'filter'`) so only `transform` animates — keep that if you touch the zoom, and don't reintroduce a per-frame `filter`/`box-shadow` on an element GSAP scales.
- The bottle webp images in `public/img/` were cut out with a Python flood-fill from the jpgs in `references/`; those source jpgs have edge artifacts and a white-background reflection that must be removed if you regenerate them.
